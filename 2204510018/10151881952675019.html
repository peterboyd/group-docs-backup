Linux, Gaming and MS Windows Applications
CONTENT OF THE DOCUMENT (just copy/paste it):

Many of us come from Windows or have a dual boot Linux/Windows system.

And the main interest in such situations is if they will be able to play their favorite MS Win games, if they will be able to run their favorite MS Win applications and so on in their Linux.


It is not an easy answer to give, but some general "rules" apply...

  • If you expect your MS win favorite game to work ok and this game is a very recent one having all the candies of DirectX etc... you may be disappointed.
  • Common applications and games like MS Office, IE explorer etc, usually work well in Linux.
  • Applications for controlling your mobile device (using vendor specific software like Samsung's Kies etc) usually do not work because of driver issues under emulation (this includes even VirtualBox and family). The good thing is that there are plenty of native Linux apps that do the job well.
  • There are some games that have their Linux binaries too, so you may look for a Linux binary in the download section of the main site of your game. Examples: Second Life, Minecraft (it's a Java app infact) etc

  • There are pretty much Linux native games, but do not expect too much from them in terms of graphics... consult the -> http://www.lgdb.org and -> http://en.wikipedia.org/wiki/Linux_gaming

WINE

====

Wine ( http://www.winehq.org ) lets you run Windows software on other operating systems.


With Wine, you can install and run these applications just like you would in Windows. Wine is still under active development so not every program works yet. But consult the Database of Wine if your application is supported -> http://appdb.winehq.org/

Remember: always keep your Wine binaries latest version, so the chance of running the app of your choice to be greater.


http://www.pcworld.com/article/2025909/gaming-on-linux-a-guide-for-sane-people-with-limited-patience.html



WINETRICKS

=========

Winetricks (http://wiki.winehq.org/winetricks) is a GUI application to help with configuring Wine a little easier. Actually it is a helper script to download and install various redistributable runtime libraries needed to run some programs in Wine. These may include replacements for components of Wine using closed source libraries. Winetricks can simply install applications to their own prefix so that if you want to run different programs under different versions of wine as they may work better that way, then having them installed in their own environment is preferred. Winetricks is known for its ease of managing wine and installing software and is supported on winehq whereas playonlinux isn't.



WINE: PLAY ON LINUX

================

Play on Linux (http://www.playonlinux.com/en) is actually based on Wine. It offers you some 'scripts' to let you install lots of games and other windows software more easily in Wine. What these scripts do:

  • Make a sort of 'bottle' to install the best wine version for the specific game or program you want to install, so all your Wine installations will be separated from each other. (This is due of the fact that some games or drivers may affect other games or programs in the same Wine installation.)
  • Installs the needed drivers, components or fonts.
  • Installs your game or program by downloading it from the Internet, using a CD or using an .exe file
  • Makes shortcuts
  • Allows you to easily configure settings without having to use a command line.


Note: if you remove a 'bottle' in Play on Linux, the whole wine installation in that bottle, with its game and drivers will be removed. Other wine installations will be unaffected.



To fully understand how Play on Linux works and how you can install windows software with it, read its the documentation --> http://www.playonlinux.com/en/page-2-Documentation.html



Here is a list off all the windows games and software, known to work with Play on Linux --> http://www.playonlinux.com/repository/?cat=1&type=0



VIRTUALBOX

=========

VirtualBox ( https://www.virtualbox.org/ ) is a powerful x86 and AMD64/Intel64 virtualization product for enterprise as well as home use. It is an extremely feature rich and high performance product.

Actually you may look at it as your virtual brand new PC box, that needs an OS to be installed. This means you need a Windows/Linux/Mac installation media, that you will use to install your OS on the virtual PC. This virtual PC will actually run in your Linux... Thus the bad side of this is that it requires more resources i.e. a powerful  real PC.

Another bad side - do not expect games to work ok.. the 3D support is there but still not that mature...

More detailed instructions on how to install Windows in a Linux VirtualBox environment you can find here -> http://www.knowliz.com/2009/01/install-windows-7-with-ubuntu-using.html* Note : you could run a native partition as a virtual machine (your windows partition if you use a dual-boot)


For Ubuntu family (as in Debian and the main RPM distributions) follow these steps for installing VirtualBox (VB):

  1. Add the VB repo as described here -> https://www.virtualbox.org/wiki/Linux_Downloads (look in the section "Debian-based Linux distributions" or the "RPM-based Linux distributions" section depending on your distribution)
  2. In your package manager select the latest VB package (not the OSE one) and install it.
  3. Go to https://www.virtualbox.org/wiki/Downloads and Download the Oracle VM VirtualBox Extension Pack that should be the same version number as your VB that you have installed from your package manager
  4. Click on the downloaded Extension pack file (alternatively you can install it from the Options menu of VB itself) so it can be autoinstalled in your VB application.
  5. Next follow the steps described in the tutorial link above (http://www.knowliz.com/2009/01/install-windows-7-with-ubuntu-using.html)




GENERAL NOTES AND TIPS (by users):

=========================


  • I have tried many methods of running Windows games while still having Linux installed on my computer. Running VM software such as Virtual-box or others do not support hardware acceleration. All the first person shooters and action Windows games require this. Thus running Windows in a VM is not a viable alternative for running games. Another alternative is installing Wine and/or it's variations. PlayonLinux and Crossover are pre-configured versions of Wine. Many games will work fine in them. But I would recommend turning off Compiz or any of the 3D add in applications in Gnome or KDE. Running 3D in Linux will give you pretty effects, but will compete with Wine for resources. Turn them off in Linux first. If you must play Windows games, it is best to install Windows on your computer also. If you only have 1 drive installed, it is best to create some free space on it and then install Linux in the free space. Your favorite distribution should then install a boot-loader such as Grub. This will ask you which operating system you wish to run on boot up. I have Linux installed on a completely different drive than Windows. When I installed Linux, I disconnected my Windows drive and installed it on a new drive. I do not use a boot loader either. I tell my BIOS which drive to start first. I believe this is the simplest and safest way to run Linux. I actually have 3 drives installed. I use one for Windows, one for Mint, and one for experimenting.
