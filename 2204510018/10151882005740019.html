General Purpose Linux Links Every Penguinisto Should Know
CONTENT OF THE DOCUMENT (just copy/paste it):

This is a page for general links which will work in any distribution. It assumes you've already got Linux installed and you know about general care and feeding of your distribution of choice. 

 

GETTING HELP: Have a question? Type "man subject" first on the command line. Q to quit. Replace subject with what you're interested in suchas "man apache" or "man cron". --help also produces the expected set of results. 

 

 

BASH programming intro: http://tldp.org/HOWTO/Bash-Prog-Intro-HOWTO.html

BASH is the general purpose Linux shell which has become the standard of the community. If you're coming from ksh or ash, it will feel similar. You can make your life a lot easier if you learn how to script one offs on the command line.

 

 

PERL programming intro: http://perldoc.perl.org/perlintro.html

PERL became the crossover language for a lot of people (including myself) where we had to write-once-run-everywhere among disparate Linux systems. Perl is available on almost all *NIX similar boxes. If it's not written in BASH, it's probably written in PERL. (If you're wondering "Can I script for a Mac? Look for perl).

 

 

Look.org: http://kde-look.org/ and http://gnome-look.org/

Think someone's theme is cool? Check out these links to pimp your penguin.

 

 

Editing Files: http://www.eng.hawaii.edu/Tutor/vi.html OR http://www.gnu.org/software/emacs/

I prefer vi, but discussions on this topic is the stuff of legend. Both vi and emacs are installed in major *NIX systems. Pick one, learn it and love it.

 

 

What's the code look like OR where did all this come from: http://www.gentoo.org/

The last living source distro with any serious community (slackware - you break my heart). Want to tune the distro to 11? Read up. Think building a kernel is kool? You can probably learn here. Glutton for punishment or simply want to have compiler races? Great! Think Linux would run awesome on your ATARI 800? Awesome!

 

You have a problem that you cannot fix?: Try to get help from http://www.linuxquestions.org/ or from your distro's main site forums section...
