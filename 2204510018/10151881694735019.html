Linux distributions compared
CONTENT OF THE DOCUMENT (just copy/paste it):

In this doc, we'll try to answer one very classic Linux question: which distribution should I choose? This question is rather little bit complicated as there is no best distribution for everyone, but we believe you can profit from our quick summary and comparison of well known Linux distros here.

 

Before you proceed, you should know that "the linux" inside all distributions is basically the same. What makes the difference is package management, default desktop environment, frequency of updates, default software collection, and finally, the level of user's knowledge to use the distribution.

 

Most available distributions are different in the aspects above, so generally - you are always moving in same sandbox, just with different preferences.

 

 

 

Arch Linux (Beginners: ease of use 3/5 | ease of installation: 2/5):

  • Package manager: pacman (binary)
  • Default desktop environment: none
  • Documentation: good
  • Community modified kernel: yes
  • Installation in: shell
  • Additional package repos: AUR - User driven package repository, with possibility to add your own package (or version of already existing package).

 

PROS:

  • Usage of vanilla software versions (eg. installed in default locations).
  • Methodical package names. (eg. you can possibly guess package's name)
  • Advanced package manager. you can list package files, find package which owns file in filesystem, search for exact file in packages, etc.
  • Small default collection of software - no need to uninstall loads of unnecessary applications.

 

Aimed towards more experienced users, Arch Linux provides a simple,   lightweight distribution for any purpose. Arch uses BSD-style init scripts.

 

 

 

Gentoo linux (Beginners: ease of use 3/5 | ease of installation: 1/5):

  • Package manager: emerge/portage (source)
  • Default desktop environment: none
  • Documentation: good
  • Community modified kernel: yes
  • Installation in: shell
  • Additional package repos: Overlays - custom package trees -> http://overlays.gentoo.org/ , ability for user's local overlay

 

PROS:

  • Absolute customization and control over the system
  • Advanced package manager
  • Methodical package names. (eg. you can possibly guess package's name)
  • USE flags: implementation of "--with-" argument from compilation process (anything you don't list in USE flags won't be included)

 

CONS:

  • Easy to mess up whole system
  • Long installation process compared to distros using only binary packages

 

Gentoo is distribution for experienced users who want absolute control over the system. It allows one to exactly specify what applications will be installed, and what features should they support (eg. image viewer-png,jpg,...). By default, it includes only most needed utilities to perform basic installation.

 

 

 

Pardus linux (Beginners: ease of use 4/5 | ease of installation: 5/5):

  • Package manager: Pisi 
  • Central configuration GUI: Pisi-Package Manager
  • Package Type: .pisi
  • Default desktop environment: KDE (gnome 3 possible)
  • Documentation: Middle
  • Community modified kernel: yes
  • Installation in: Yali installation manager

 

PROS:

  • Very stable and integrated KDE environment
  • Comar makes PC faster and lets programs interact better with each other
  • Has its own integrated software
  • Great hardware support and configuration manager for video drivers
  • Kaptan welcomes very user friendly the new users at your first login

 

CONTRAS:

  • Small repo's
  • Pardus sometimes has a hard time with the API's of some Laptop models
  • Not so well-known --> less support

 

Pardus is a Linux distro that fights for its freedom. Unlike most other distro's, its not based upon another Linux Distro. This makes Pardus very unique and different. With Pardus, comes a lot of pre installed codecs and software, which makes it an good beginners Linux distro.

 

 

 

Ubuntu Family (Beginners: ease of use 4/5 | ease of installation: 5/5):

  • Package manager: Software Center
  • Package type: .deb
  • Default desktop environment: depends on the family member (GNOME, KDE etc)
  • Documentation: perfect
  • Community modified kernel: yes
  • Installation in: GUI
  • Additional package repos: Personal Package Archives (PPA), GetDeb repos -> http://www.getdeb.net/welcome/

 

PROS:

  • Good and easy control over the system configuration
  • Perfectly documented and supported
  • A huge number of packages available, most of them recent versions
  • Very user oriented and preferred by novice users
  • The different family members are faced towards easing the user with their choice depending on if they like more KDE (Kubuntu), LXDE (Lubuntu), XFCE (Xubuntu), server edition etc -> http://en.wikipedia.org/wiki/List_of_Linux_distributions#Ubuntu-based

 

CONS:

  • Video driver support is by default to open source drivers. Additional steps are needed to add most recent propriety graphic drivers.
  • Some propriety codecs/apps are not included by default, so additional steps for using them are needed (like libdvdcss2 for example)
  • In the latest releases (above 11.04) a lot of very recent versions applications are included, introducing serious bugs and speed penalties (incl. Unity)

 

DISPUTABLE:

  • Often version releases. The good thing about them are that you have always almost the latest versions of the different packages including the base ones like KDE, GNOME, X etc... The bad side is that sometimes these latest versions are still buggy at the time of the release, so at the moment of release sometimes it may contain bugs.
  • Unity - the ease of use and the user friendliness of Unity is disliked by a lot of users. It changes the habits of old users and they claim it is more suitable for tablets. But of course Unity has its fans.

 

Ubuntu family consists of Debian based distributions, that are aimed at users' ease of use, flexibility, recent packages and perfect support. It has a huge number of users, thus making the Ubuntu family a very well supported one. The different family members are faced towards easing the user with their choice depending on if they like more KDE (Kubuntu), LXDE (Lubuntu), XFCE (Xubuntu), server edition etc -> http://en.wikipedia.org/wiki/List_of_Linux_distributions#Ubuntu-based. Great for novice users.
