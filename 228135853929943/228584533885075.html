TITLE: Linux, Gaming and MS Windows Applications

Many of us come from Windows or have a dual boot Linux/Windows system.

And the main interest in such situations is if they will be able to play their favorite MS Win games, if they will be able to run their favorite MS Win applications and so on in their Linux.

It is not an easy answer to give, but some general "rules" apply...

  • If you expect your MS win favorite game to work ok and this game is a very recent one having all the candies of DirectX etc... you may be disappointed.
  • Common applications and games like MS Office, IE explorer etc, usually work well in Linux.
  • Applications for controlling your mobile device (using vendor specific software like Samsung's Kies etc) usually do not work because of driver issues under emulation (this includes even VirtualBox and family). The good thing is that there are plenty of native Linux apps that do the job well.
  • There are some games that have their Linux binaries too, so you may look for a Linux binary in the download section of the main site of your game. Examples: Second Life, Minecraft (it's a Java app infact) etc
  • There are pretty much Linux native games, but do not expect too much from them in terms of graphics... consult the -> http://www.lgdb.org and -> http://en.wikipedia.org/wiki/Linux_gaming

For running all your MS Win applications you may use:

WINE

====

Wine ( http://www.winehq.org ) lets you run Windows software on other operating systems.

With Wine, you can install and run these applications just like you would in Windows. Wine is still under active development so not every program works yet. But consult the Database of Wine if your application is supported -> http://appdb.winehq.org/

Remember: always keep your Wine binaries latest version, so the chance of running the app of your choice to be greater.

WINE: PLAY ON LINUX

===================

Play on Linux (http://www.playonlinux.com/en) its Basically Based on wine. It offers you some 'scripts' to let you install lots of games and other windows software more easily in wine. what these scripts do:

  • Make a sort of 'bottle' to install the best wine version for the specific game or program you want to install, so all your wine installations will be separated from each other. (This is due because of the fact that some games or drivers may affect other games or programs in the same wine Installation.)
  • Installs needed drivers, components or fonts.
  • Installs your game or program by downloading it from the internet, using a CD or using an .exe file
  • Makes shortcuts

Note: if you remove a 'bottle' in Play on Linux, The whole wine installation in that bottle, with its game and drivers will be removed. Other wine installations will be unaffected.

To fully understand how Play on Linux works and how you can install windows software with it, is written in the documentation --> http://www.playonlinux.com/en/page-2-Documentation.html

VIRTUALBOX

=========

VirtualBox ( https://www.virtualbox.org/ ) is a powerful x86 and AMD64/Intel64 virtualization product for enterprise as well as home use. It is an extremely feature rich and high performance product.

Actually you may look at it as your virtual brand new PC box, that needs an OS to be installed. This means you need a Windows/Linux/Mac installation media, that you will use to install your OS on the virtual PC. This virtual PC will actually run in your Linux... Thus the bad side of this is that it requires more resources i.e. a powerful real PC.

Another bad side - do not expect games to work ok.. the 3D support is there but still not that mature...

More detailed instructions on how to install Windows in a Linux VirtualBox environment you can find here -> http://www.knowliz.com/2009/01/install-windows-7-with-ubuntu-using.html

For Ubuntu family (as in Debian and the main RPM distributions) follow these steps for installing VirtualBox (VB):

  1. Add the VB repo as described here -> https://www.virtualbox.org/wiki/Linux_Downloads (look in the section "Debian-based Linux distributions" or the "RPM-based Linux distributions" section depending on your distribution)
  2. In your package manager select the latest VB package (not the OSE one) and install it.
  3. Go to https://www.virtualbox.org/wiki/Downloads and Download the Oracle VM VirtualBox Extension Pack that should be the same version number as your VB that you have installed from your package manager
  4. Click on the downloaded Extension pack file (alternatively you can install it from the Options menu of VB itself) so it can be autoinstalled in your VB application.
  5. Next follow the steps described in the tutorial link above (http://www.knowliz.com/2009/01/install-windows-7-with-ubuntu-using.html)
