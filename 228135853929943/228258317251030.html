TITLE: Linux First Steps and "Rules"

A lot of people here ask questions about different aspects of Linux.

Since answering these questions again and again is a tedious work, i will try to summarize some of the important topics that may be of interest to new Linux users...

First of all if you decided to test Linux, choose you distro and download/burn its LiveCD version (if such version is offered). That will help you in your decision.

Choosing the right distro is usually hard, because you get quite a bunch of opinions which distro is better, which sucks etc...

But these links may help you choose:

Distrowatch -> http://distrowatch.com/dwres.php?resource=major

Linux Wizard -> http://www.zegeniestudios.net/ldc/index.php?select_lang=true

After you install your distribution and get a bit acquainted with it you may like to have a look at some of the following:

==============================================================

  1. Make sure you only add to your repositories trusted sources
  2. Make sure you have activated all your distro's security updates and general updates repositories
  3. You may add some additional repositories that offer new versions of some packages like KDE, some multimedia programs etc
  • Ubuntu family:

Backports - https://launchpad.net/~kubuntu-ppa and https://help.ubuntu.com/community/Repositories/Kubuntu

Medibuntu - http://medibuntu.org/repository.php

Wine - http://www.winehq.org/download/ubuntu

VirtualBox - https://www.virtualbox.org/wiki/Linux_Downloads

  • Mandriva:

The PLF Repos - http://easyurpmi.zarb.org/

  • Fedora:

Read this first -http://fedoraproject.org/wiki/Third_party_repositories

RPM Fusion - http://rpmfusion.org/Configuration

Linva (Add RPM Fusion FIRST) -http://rpm.livna.org/

  • openSuse:

Read this first -http://en.opensuse.org/Package_repositories

Packman - http://en.opensuse.org/Additional_package_repositories

Note that most repos in SuSE are available under "add repository" and selecting "Community Repository".

  • Pardus:Read this first for installation and more: http://en.pardus-wiki.org/Installation:Contents PardusRepo: http://worldforum.pardus-linux.nl/index.php?topic=3478.0 (Second post from Atobloo).

  1. Avoid installing unnecessary services. Like if you do not intend accessing your computer remotely, do not install the SSH server etc
  2. Never give your root password (or root access) to anyone you do not entirely trust
  3. Do not run scripts from unknown sources especially do not run them as root. Same goes for installing packages from untrusted sources...

Finally if you feel lost among the pile of applications that Linux offers you, and you need some help in finding alternatives/analogues of you favorite MS Windows programs.... You may consult the Linux Alternative Project (formerly the Linux Equivalent Project) -> http://www.linuxalt.com/
